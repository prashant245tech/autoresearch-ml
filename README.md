# AutoML Price Predictor

Karpathy AutoResearch pattern applied to predictive ML.
An AI agent iterates on feature engineering and model selection to find
the best predictor for your target variable (default: corrugated box Price/MSF).

---

## File Structure

```
corrugated_automl/
├── program.md          ← THE ONLY FILE YOU EDIT (plain English config + domain knowledge)
├── compile.py          ← Reads program.md → calls an OpenAI-compatible LLM → produces feature_spec.json
├── prepare.py          ← FIXED: reads feature_spec.json → cleans data → train/test split
├── train.py            ← THE ONLY research surface: description + features + model config
├── run_experiment.py   ← IMMUTABLE harness: budget, evaluation, logging, auto loop
├── requirements.txt
├── feature_spec.json   ← Auto-generated by compile.py (reviewable, editable)
├── data/
│   ├── your_data.csv   ← YOUR DATA GOES HERE (CSV or XLSX)
│   ├── train.parquet   ← Auto-generated by prepare.py
│   ├── test.parquet    ← Auto-generated by prepare.py
│   └── columns.json    ← Auto-generated metadata for train.py
├── experiments/
│   ├── results.json    ← All experiment results
│   ├── results.csv     ← Same, as CSV for easy inspection
│   └── research_state.json ← Frozen-input baseline for the current research session
└── models/
    ├── best_model.pkl  ← Auto-saved best model
    ├── best_features.pkl
    └── best_train.py   ← train.py snapshot that produced the best model
```

## Workflow

### Step 1 — Edit program.md (the only file you touch)

Three sections to fill in:

**Section 1** — data file path and target column:
```
data_file: data/your_data.xlsx
target_column: Tier 1 Price/MSF
```

**Section 2** — describe your features in plain English:
```
Size Bucket:
  Ordered size band for the box. Unknown values get -1.

Ink Coverage Bucket:
  Ordered print coverage bucket: 0-10%, 10-20%, ... , >50%.
  Unknown values get -1.
```

**Section 3** — domain context for the AI agent (what features to try, pricing logic).

### Step 2 — Compile (once per program.md change)

```bash
export OPENAI_API_KEY=sk-...
python compile.py
```

The compiler reads your plain English descriptions and produces `feature_spec.json`
with exact cleaning rules (null handling, encoding, clipping, consolidation).
Review the printed summary — edit `feature_spec.json` directly if anything looks wrong.

You can also put provider settings in a project-local `.env` file:

```bash
OPENAI_API_KEY=sk-...
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4.1-mini
```

To target another OpenAI-compatible backend, also set:

```bash
export OPENAI_BASE_URL=https://your-llm-endpoint/v1
export OPENAI_MODEL=qwen/qwen3-32b
```

`OPENAI_MODEL` is the shared default. You can also use `COMPILE_LLM_MODEL`
or `AUTORESEARCH_LLM_MODEL` to override the compiler and autonomous agent separately.

### Step 3 — Prepare data (once per data change)

```bash
python prepare.py
```

Executes `feature_spec.json` deterministically — no API calls.
Produces `data/train.parquet`, `data/test.parquet`, `data/columns.json`.

### Step 4a — Manual experiments

Edit only these hooks in `train.py`, then run the immutable harness:
- `EXPERIMENT_DESCRIPTION`
- `engineer_features(df, meta)`
- `get_model_config()`

```bash
python train.py                           # Run one experiment
python run_experiment.py --summary        # See all results
python run_experiment.py --best           # See best model details
```

Every run uses the same fixed 5-minute experiment budget.

### Step 4b — Autonomous agent mode

```bash
git init
git add .
git commit -m "Initial snapshot"
python run_experiment.py --n 10 --auto
```

The agent iterates automatically: reads experiment history + frozen metadata →
proposes a new `train.py` → the harness validates that only the approved
research surface changed → commits the candidate → runs a 5-minute experiment →
keeps the commit if MAPE improves or resets `train.py` if it does not.
Autonomous mode requires a git repository so accepted changes are preserved and
rejected changes can be discarded safely.

### Research Session Rules

- `compile.py`, `feature_spec.json`, `prepare.py`, `data/train.parquet`,
  `data/test.parquet`, and `data/columns.json` are frozen for a research session.
- The first harness run snapshots those inputs in `experiments/research_state.json`.
- If any frozen input changes later, the harness stops and asks you to rerun setup
  and start a fresh research session.
- Only `train.py` is mutable during research.

## To Use With a Different Dataset

1. Update `data_file` and `target_column` in `program.md` Section 1
2. Rewrite Section 2 with your new feature descriptions
3. Update Section 3 with domain context for the agent
4. Run `compile.py` → `prepare.py`
5. Delete `experiments/research_state.json` if it exists
6. Update `train.py` only if you want to try new research ideas
7. Run `train.py` or `run_experiment.py`

No code changes are needed in `prepare.py` or `run_experiment.py`.

## Performance Targets (corrugated pricing)

| MAPE    | Assessment        |
|---------|-------------------|
| > 20%   | Poor              |
| 15–20%  | Needs improvement |
| 10–15%  | Acceptable        |
| 5–10%   | Good              |
| < 5%    | Excellent         |
